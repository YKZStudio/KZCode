---
title: 引用
description: 将本地目录和 Git 仓库添加为项目引用。
---

引用让 OpenCode 能够访问当前项目之外的目录。你可以借助引用，在工作时使用文档、共享库、示例或其他仓库。

引用通过别名在 `opencode.json` 或 `opencode.jsonc` 中进行配置。

```jsonc title="opencode.jsonc"
{
  "$schema": "https://opencode.ai/config.json",
  "references": {
    "docs": {
      "path": "../product-docs",
      "description": "用于了解产品行为和文档规范",
    },
    "sdk": {
      "repository": "anomalyco/opencode-sdk-js",
      "branch": "main",
      "description": "用于了解 JavaScript SDK 的实现细节",
    },
  },
}
```

---

## 本地目录

使用 `path` 引用一个本地目录。

```jsonc title="opencode.jsonc"
{
  "references": {
    "docs": {
      "path": "../docs",
    },
  },
}
```

路径可以是：

- 相对于定义该引用的配置文件的路径
- 绝对路径，例如 `/home/user/docs`
- 相对于你的主目录的路径，例如 `~/docs`

你也可以使用字符串简写形式：

```jsonc title="opencode.jsonc"
{
  "references": {
    "docs": "../docs",
  },
}
```

## Git 仓库

使用 `repository` 引用一个 Git 仓库。OpenCode 会在其本地仓库缓存中实例化该仓库，并将检出的源代码作为引用目录提供给你使用。

```jsonc title="opencode.jsonc"
{
  "references": {
    "effect": {
      "repository": "Effect-TS/effect",
      "branch": "main",
    },
  },
}
```

`repository` 接受 Git URL、host/path 形式的引用，以及 GitHub `owner/repo` 简写形式。可选的 `branch` 字段用于选择某个分支或 ref。如果不指定 `branch`，OpenCode 会使用该仓库的默认分支。

当你不需要指定分支、描述或其他选项时，可以使用字符串简写形式：

```jsonc title="opencode.jsonc"
{
  "references": {
    "effect": "Effect-TS/effect",
  },
}
```

:::note
Git 引用会以异步方式刷新。新配置的仓库可能需要稍等片刻才能完成克隆或更新。
:::

## 描述用途

添加 `description` 以说明代理应在何时使用某个引用。

```jsonc title="opencode.jsonc"
{
  "references": {
    "design-system": {
      "path": "../design-system",
      "description": "在实现 UI 组件或设计令牌时使用",
    },
  },
}
```

OpenCode 会将带有描述的引用纳入代理上下文。描述应当简短且足够具体，以便区分内容相似的引用。没有描述的引用仍可通过自动补全和直接使用来访问，但不会主动告知代理。

---

## 隐藏自动补全条目

将 `hidden` 设为 `true`，可在 TUI 的 `@` 自动补全中省略某个引用。

```jsonc title="opencode.jsonc"
{
  "references": {
    "internal": {
      "path": "../internal",
      "description": "用于了解内部实现细节",
      "hidden": true,
    },
  },
}
```

`hidden` 仅影响自动补全。带有描述的隐藏引用仍会被纳入代理上下文。

## 使用引用

已配置的引用会出现在 TUI 的 `@` 自动补全中。输入 `@别名` 可附加该引用的根目录，输入 `@别名/` 可在其内部搜索文件。

```text
将此实现与 @sdk/src/client.ts 进行比较
```

代理还会在其系统上下文中收到已配置且带有描述的引用的解析路径和描述，因此在相关时，代理无需你手动附加即可查看该引用。

OpenCode 会自动放行引用目录，使其通过外部目录权限边界。常规的工具权限仍然适用；例如，无法编辑文件的代理不会因为某个目录被配置为引用就获得编辑权限。

## 配置字段

| 字段          | 本地 | Git | 说明                                             |
| ------------- | ---- | --- | ------------------------------------------------ |
| `path`        | 是   | 否  | 本地引用目录                                     |
| `repository`  | 否   | 是  | Git URL、host/path 或 GitHub `owner/repo` 值     |
| `branch`      | 否   | 是  | 可选的 Git 分支或 ref                            |
| `description` | 是   | 是  | 用于说明何时使用该引用的指引                     |
| `hidden`      | 是   | 是  | 在 TUI 的 `@` 自动补全中隐藏该引用               |

引用别名不能为空，也不能包含 `/`、空白字符、反引号或逗号。
